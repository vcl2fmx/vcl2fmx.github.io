Conversion Contracts in v5.0

Version 5.0 adds an executable conversion-contract system. A contract is a small Delphi sample input paired with an expectation file. The expectation records the required output, report behavior, uses-clause result, and validation rules for one known converter behavior.

The contract runner is the engineering test utility that finds each expectation file, runs the converter against the matching sample input, and fails if the generated result differs from the expected result. It is not runtime data for normal user conversions.

Contract AreaWhat It Protects

Graphics and GDIFMX canvas substitutions where reliable, plus visual-review reporting where drawing still needs inspection.

The current v5.0 contract suite contains 194 executable expectations. The current verification run passed with 194 passing and 0 failing contracts.
