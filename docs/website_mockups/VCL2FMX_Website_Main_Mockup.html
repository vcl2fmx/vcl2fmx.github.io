Free open-source Delphi migration utility

Convert VCL projects toward FireMonkey with clear reports and editable mapping packs.

VCL2FMXConverter helps Delphi developers move VCL projects toward FMX by converting many common forms, components, properties, events, and project structures into FireMonkey equivalents.

The converter is designed to reduce repetitive migration work and produce a strong FMX starting point. Items that cannot be converted safely are reported for manual review instead of being hidden.

What the converter does

The project focuses on practical VCL-to-FMX migration assistance, not unrealistic one-click conversion. It converts what it can handle safely and documents the rest.

Forms and components

Converts many standard VCL visual controls to FMX equivalents and preserves layout information where practical.

Properties and events

Maps common VCL properties and events into FMX names and structures when the conversion is safe.

Reports and review

Generates readable reports for unsupported APIs, third-party controls, manual review items, and conversion risks.

Mapping packs

Mapping packs are optional JSON rule files that let users inspect, modify, and extend third-party component support.

Transparent rules

A mapping pack can convert simple components, partially convert components with warnings, detect complex controls, or preserve selected references when appropriate.

Community-extensible

Developers can create or adjust packs for component libraries they own and understand, without forcing unsafe assumptions into the converter core.

Known limits

This section should stay visible. It makes the project more trustworthy and reduces unrealistic expectations.

No automated converter can safely convert every VCL application to FMX. VCL and FMX differ in rendering, Windows message handling, data binding, platform behavior, and component libraries. VCL2FMXConverter converts what it can safely handle and reports uncertain or unsafe items for manual review.

Contact and feedback

Bug reports, test results, and suggestions are welcome. Please include the converter version, Delphi version, and a short description of the project or error.
