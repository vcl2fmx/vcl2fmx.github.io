Suggested Site Structure

A calmer, smaller technical site for a free Delphi utility. This avoids a commercial sales-page feel and gives developers the information they need quickly.

Recommended top-level pages or sections

Home

Short explanation, current version, primary download, key conversion areas, and honest limitations.

Downloads

Stable source ZIP, guides, mapping packs, optional tools, and license. Keep this page very clear.

Mapping Packs

Explain JSON packs, included vendors, rule actions, and how users can modify or contribute packs.

Documentation

User Guide, Engineering Guide, rules references, and conversion-report examples.

Release Notes

Plain version history. Make the current stable release obvious and avoid mixing stable and development versions.

Contact

Bug reports, test results, suggestions, and the information users should include when reporting problems.

What to keep and what to remove

Keep

  • Free and open-source message
  • Current stable version and download button
  • Mapping-pack explanation
  • User Guide and Engineering Guide
  • Honest limitations and manual-review expectations
  • Contact instructions for useful bug reports

Remove or soften

  • Old v4.0 references
  • "Up to 95%" conversion claims
  • Repeated marketing phrases
  • Large commercial-style hero language
  • Duplicate download links
  • Anything implying guaranteed full conversion
