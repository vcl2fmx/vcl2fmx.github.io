# Security policy

## Supported code

Security fixes are applied to the current v5.0 code line. Older source snapshots and generated distribution archives may not receive separate fixes.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could expose users, projects, source code, or local files.

Report the issue privately by email to `churchesite@gmail.com` with:

- the VCL2FMXConverter version or commit;
- the Delphi and Windows versions;
- a concise description of the risk;
- steps or a minimal non-confidential fixture that reproduces it; and
- any known workaround.

Do not attach proprietary project source or sensitive data. A reduced fixture is preferred.

The project maintainer will review the report, confirm whether it is reproducible, and coordinate a fix and disclosure when appropriate.
